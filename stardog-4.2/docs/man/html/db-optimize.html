stardog-admin db optimize Manual Page

NAME


stardog-admin db optimize —Optimizes the disk format of a disk database

SYNOPSIS


stardog-admin [ --server <server url> ]db optimize [ {-p | --passwd} <password> ] [ {-P | --ask-password} ] [ {-u | --username} <username> ] [ {-v | --verbose} ] [--] <database>

OPTIONS


-p <password>, --passwd <password>
Password.
-P, --ask-password
Prompt for password.
--server <server url>
URL of Stardog Server. If this option isn't specified, it will be read from JVM argument 'stardog.default.cli.server'. If the JVM arg isn't set, the default value 'http://localhost:5820' is used. If server URL has no explicit port value, the default port value '5820' is used. Example: 'stardog-admin --server http://12.34.56.78:5820 server stop'
-u <username>, --username <username>
User name.
-v, --verbose
Verbose flag that causes more detailed information to be printed
--
This option can be used to separate command-line options from the list of argument, (useful when arguments might be mistaken for command-line options
<database>
The name of the database for the operation

DISCUSSION


Optimizes disk indexes for query answering after a database has been heavily modified. Stardog automatically optimizes a database once a certain number of modifications have been made. This command can be used to eagerly optimize a database. Calling this may be a no-op if Stardog has recently performed the optimization. This optimization does not change any database contents.

EXAMPLES


Optimize the database 'myDb':

    $ stardog-admin db optimize myDb
